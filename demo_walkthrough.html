Provenance Guard: Demo Script

target: about 2 minutes

Read this while you record your screen. You are showing the program we built, running in your terminal. Each scene has three parts: Do (run the command), Say (read the line), and See (what should appear). If a live result lands in a slightly different band, say so out loud. The system is built to show uncertainty.

Before you record

1

What it is

~20 sec
Say
This is Provenance Guard, a backend I built. A platform sends it text. It returns whether the text reads as human or AI, a confidence score, and a label to show readers. Creators can appeal a result, and every decision is logged. I will show it running.
Do (check the service is up)
See: status: ok, the audit log reachable, and the Groq key present.

A high-confidence result

~30 sec
Do (submit a repetitive paragraph)
Say
This comes back as likely AI, with confidence around 0.88. Two signals run on the text. One is a language model that judges meaning. The other is a set of stylometry measurements that judge structure, like how much the sentence lengths vary. Here both point the same way, so the confidence is high.
See: attribution: likely_ai, confidence near 0.88, and the AI label text. Copy the content_id from the response and paste it into the Content ID box above. You need it in Scene 4.

A low-confidence result

~30 sec
Do (submit a short note)
Say
This comes back uncertain, around 0.58. It is only seven words, and the two signals disagree, so the score is low. Short text is never labeled AI, because calling a person's work AI is the worst mistake the system can make. The score changes with the evidence, so 0.88 and 0.58 give different labels.
See: attribution: uncertain, confidence near 0.58, and the uncertain label.
4

An appeal

A creator can appeal a result. They send the content ID and their reason. The status changes to under review. The system does not re-score it automatically. It records the appeal next to the original decision so a person can look at it.

The audit log

Close

~15 sec

That is the full flow: two signals, a confidence score, a plain label, an appeal path, and an audit log. The design decisions and the evidence are written up in the README.
Optional rate-limit demo, only if you have time. It sends 12 quick submits. The first ten return 200, the last two return 429. Note that this makes real model calls, so it runs slowly.

+

Design points you can mention if there is time

  • Two signals. One reads meaning (the language model), one reads structure (the measurements). They can disagree, and disagreement lowers the confidence.
  • A score, not yes or no. AI detection is not reliable, so a score that shows uncertainty is more honest than a forced answer.
  • The AI side is stricter. Calling a person's work AI is the worst mistake, so the AI cutoff is 0.80 and the human cutoff is 0.70. Short text is never labeled AI.
  • Appeals. Some results will be wrong. An appeal lets the creator contest it, and the content is marked under review.
  • Audit log. Every decision and appeal is recorded, so results can be checked later.
